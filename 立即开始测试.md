# 🚀 立即开始测试 - 详细步骤

## 📋 准备工作（5分钟）

### 第一步：重新编译硬件固件（2分钟）

#### 1.1 打开Keil MDK
- 双击桌面上的Keil图标
- 或者从开始菜单找到"Keil uVision5"

#### 1.2 打开项目
```
文件路径：
f4_26_1.1/f4_26_1.1/f4_26_1.1/MDK-ARM/f4.uvprojx

操作：
1. 点击菜单：Project → Open Project
2. 导航到上面的路径
3. 选择f4.uvprojx
4. 点击"打开"
```

#### 1.3 编译项目
```
操作：
1. 按F7键（或点击菜单：Project → Build Target）
2. 等待编译完成（约30秒）
3. 查看输出窗口，确认显示：
   "0 Error(s), 0 Warning(s)"
   或
   "Build succeeded"
```

#### 1.4 下载到硬件
```
操作：
1. 确保硬件已连接到电脑（USB或ST-Link）
2. 确保硬件已上电
3. 按F8键（或点击菜单：Flash → Download）
4. 等待下载完成（约10秒）
5. 查看输出窗口，确认显示：
   "Programming Done"
   或
   "Download verified successfully"
```

#### 1.5 重启硬件
```
操作：
1. 断开硬件电源
2. 等待3秒
3. 重新上电
4. 硬件应该正常启动
```

### 第二步：重新编译APP（3分钟）

#### 2.1 打开命令行
```
Windows：
1. 按Win+R
2. 输入cmd
3. 按回车

或者：
1. 在文件夹中按Shift+右键
2. 选择"在此处打开命令窗口"
```

#### 2.2 进入项目目录
```bash
cd RideWind
```

#### 2.3 编译并运行APP
```bash
flutter run
```

#### 2.4 等待编译完成
```
输出信息：
1. "Running Gradle task 'assembleDebug'..."
2. "Built build\app\outputs\flutter-apk\app-debug.apk"
3. "Installing build\app\outputs\flutter-apk\app.apk..."
4. "Syncing files to device..."
5. "Flutter run key commands."

时间：约2-3分钟
```

#### 2.5 确认APP已安装
```
手机上：
1. 查看是否有新的RideWind图标
2. 或者APP自动打开
3. 确认版本已更新
```

## 🔍 故障排查

### 问题1：编译失败

#### Keil编译错误
```
错误信息：
"error: expected expression"
"error: undeclared identifier"

解决方案：
1. 检查logo.c是否正确修改
2. 检查是否有语法错误
3. 重新复制代码
4. 清理项目后重新编译
```

#### Flutter编译错误
```
错误信息：
"Error: Conditions must have a static type of 'bool'"
"Error: The method 'xxx' isn't defined"

解决方案：
1. 检查simple_logo_uploader.dart是否正确修改
2. 运行flutter clean
3. 运行flutter pub get
4. 重新运行flutter run
```

### 问题2：下载失败

#### 硬件连接问题
```
错误信息：
"No target connected"
"Cannot access target"

解决方案：
1. 检查USB连接
2. 检查ST-Link驱动
3. 检查硬件供电
4. 重新连接后再试
```

### 问题3：上传失败

#### 蓝牙连接问题
```
症状：
- 无法连接蓝牙
- 连接后立即断开
- 发送命令无响应

解决方案：
1. 重启蓝牙
2. 重新配对设备
3. 检查蓝牙权限
4. 重启手机和硬件
```

#### 传输超时
```
症状：
- 进度卡在某个位置
- 长时间无响应
- 显示"LOGO_BUSY"

解决方案：
1. 等待30秒
2. 如果还是卡住，取消上传
3. 重新连接蓝牙
4. 再次尝试上传
```
