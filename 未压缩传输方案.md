## 🔒 数据可靠性详解

### 1. CRC32校验（最重要）
```dart
// 计算整个数据的CRC32
final crc32 = calculateCRC32(data);

// 硬件端验证
uint32_t calculatedCRC = CRC32_CalculateFlash(addr, size);
if (calculatedCRC != expected_crc) {
    return LOGO_FAIL_CRC;  // 校验失败
}
```

**即使有1个字节错误，CRC32也会检测到！**

### 2. 包序号检查
```dart
// APP端按序发送
for (int seq = 0; seq < totalPackets; seq++) {
    await sendPacket(seq, data);
}

// 硬件端检查序号
if (packet.seq != expected_seq) {
    return ERROR;  // 序号错误
}
```

**确保包的顺序正确！**

### 3. ACK确认机制
```dart
// 每30包等待ACK
if ((seq + 1) % 30 == 0) {
    final ack = await waitForACK();
    if (ack == 'TIMEOUT') {
        // 重传
        seq--;
        continue;
    }
}
```

**确保硬件收到了数据！**

### 4. 大小校验
```c
// 硬件端检查接收大小
if (logo_received_size != logo_total_size) {
    sprintf(response, "LOGO_FAIL:SIZE:%lu/%lu\n", 
            received, expected);
    return FAIL;
}
```

**确保数据完整！**

### 5. 最终校验流程
```
1. 传输完成
2. 计算接收数据的CRC32
3. 对比APP发送的CRC32
4. 如果不匹配 → 失败
5. 如果匹配 → 成功
```

**多重保护，确保100%可靠！**
