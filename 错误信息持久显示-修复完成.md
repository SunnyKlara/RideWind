# Logo上传错误信息持久显示 - 修复完成

## 问题
- Logo上传完成后，LCD显示错误信息（SIZE ERROR 或 CRC ERROR）
- 用户发送 `UI:6` 命令查看调试界面时，错误信息被清除
- 无法看到具体的错误类型和详细信息

## 根本原因
xuanniu.c 的 UI6 处理代码会清空屏幕并重新显示Logo，覆盖了错误信息

## 解决方案

### 1. 添加错误显示标志
在 logo.c 中添加全局变量：
```c
static bool logo_error_displayed = false;
```

### 2. 错误发生时设置标志
在 LOGO_END 处理中，当检测到错误时：
- SIZE ERROR: 设置 `logo_error_displayed = true`
- CRC ERROR: 设置 `logo_error_displayed = true`
- 在LCD上显示 "Press button to clear" 提示

### 3. 修改UI6界面逻辑
xuanniu.c 中的 UI6 处理：
- 检查 `Logo_IsErrorDisplayed()`
- 如果有错误显示，**不清除屏幕**，保持错误信息
- 按下按钮时清除错误标志，重新初始化界面

### 4. 新上传时清除标志
在 LOGO_START 和 LOGO_START_COMPRESSED 开始时：
- 清除 `logo_error_displayed = false`
- 允许新的上传覆盖旧的错误

## 测试步骤

1. **编译并烧录固件**
2. **运行上传测试**
3. **上传完成后，LCD会显示错误信息**：
   - 如果是 SIZE ERROR: 显示接收大小 vs 预期大小
   - 如果是 CRC ERROR: 显示预期CRC vs 实际CRC，以及前16字节数据
4. **发送 `UI:6` 命令** - 错误信息保持显示
5. **按下按钮** - 清除错误，显示正常Logo调试界面

## 预期结果
- 错误信息会持久显示在LCD上
- 用户可以清楚看到是 SIZE ERROR 还是 CRC ERROR
- 可以看到详细的错误数据用于诊断
- 按下按钮可以手动清除错误

## 下一步
根据LCD显示的错误类型：
- **SIZE ERROR**: 检查数据包丢失问题
- **CRC ERROR**: 检查APP和硬件的CRC计算是否一致
