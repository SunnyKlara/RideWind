# 紧急调试 - 检查实际发送的数据大小

## 问题
LCD显示 `Size:1024`，但代码已经修改为发送115200字节。

## 可能原因

### 1. APP没有热重载
Flutter的热重载可能没有更新这个文件。

### 2. 缓存问题
旧的编译缓存可能还在使用。

### 3. 代码路径问题
可能在其他地方还有旧的代码路径。

---

## 立即检查步骤

### 1. 检查APP日志
在APP的E2E测试界面，查看日志输出：
- 应该显示: "✓ 图片转换完成: 115200 bytes"
- 如果显示: "✓ 图片转换完成: 1024 bytes" → 代码没有更新

### 2. 完全重启APP
```bash
# 1. 停止当前运行的APP
# 2. 清理缓存
flutter clean
flutter pub get

# 3. 重新运行
flutter run
```

### 3. 检查_convertToRGB565函数
确认函数返回的数据大小：
```dart
Uint8List _convertToRGB565(img.Image image) {
  final int width = 240;
  final int height = 240;
  final result = Uint8List(width * height * 2); // 应该是115200
  // ...
  return result;
}
```

## 调试方法

### 在E2E测试中添加更多日志
在 `_startE2ETest()` 函数中：

```dart
// 1. 转换图片为RGB565格式 (240x240)
final bitmapData = _convertToRGB565(_processedImage!);
_addLog('✓ 图片转换完成: ${bitmapData.length} bytes');
_addLog('  预期数据量: 240x240x2 = 115200 bytes');

// 添加这行来确认
_addLog('  实际数据量: ${bitmapData.length} bytes');
_addLog('  匹配: ${bitmapData.length == 115200 ? "YES" : "NO"}');
```

### 检查发送的命令
在 `_sendStartCommand()` 中：

```dart
final command = 'LOGO_START:$dataSize:0\n';
_addLog('发送: $command');
_addLog('  数据大小: $dataSize bytes');

// 添加这行

```

## 解决方案

### 方案1: 完全重启（推荐）
```bash
# 停止APP
# Ctrl+C 或关闭APP

# 清理
cd RideWind
flutter clean
flutter pub get

# 重新运行
flutter run

# 重新测试
# 1. 连接设备
# 2. 进入设备连接界面
# 3. 点击"E2E测试"
# 4. 点击"开始测试"
```

### 方案2: 检查是否有其他测试界面
可能你点击的不是E2E测试界面，而是其他的Logo上传界面：
- `logo_upload_screen.dart` - 主上传界面
- `logo_upload_debug_screen.dart` - 调试界面
- `logo_upload_simple_screen.dart` - 简单上传界面
- `logo_upload_e2e_test_screen.dart` - E2E测试界面 ← 应该用这个

### 方案3: 手动验证代码
检查 `device_connect_screen.dart` 中的按钮是否指向正确的界面：

```dart
// 应该是这样
ElevatedButton(
  onPressed: () {
    Navigator.push(
      context,
      MaterialPageRoute(
        builder: (context) => const LogoUploadE2ETestScreen(),  // ← 确认这个
      ),
    );
  },
  child: const Text('E2E测试'),
),
```
