# 多Logo存储与旋钮选择功能需求

## 概述
在现有单Logo上传功能基础上，扩展支持存储最多3张自定义Logo图片，并通过硬件旋钮在Logo界面切换显示不同的已上传Logo。

## 用户故事

### US-1: 多Logo存储
作为用户，我希望能够上传并存储最多3张自定义Logo图片，以便在不同场景下使用不同的Logo。

**验收标准:**
- 1.1 硬件支持存储3个独立的Logo槽位（Slot 0, 1, 2）
- 1.2 每个槽位独立存储在Flash的不同地址区域
- 1.3 槽位地址分配：Slot 0 @ 0x100000, Slot 1 @ 0x120000, Slot 2 @ 0x140000
- 1.4 每个槽位占用128KB空间（0x20000字节）
- 1.5 上传协议向后兼容：不指定槽位时默认上传到Slot 0
- 1.6 支持通过协议指定上传目标槽位

### US-2: 旋钮切换Logo
作为用户，我希望在Logo界面（UI6）通过旋转旋钮来切换显示不同的已上传Logo。

**验收标准:**
- 2.1 在Logo界面旋转旋钮可以在有效的Logo槽位之间循环切换
- 2.2 只显示已上传有效Logo的槽位，跳过空槽位
- 2.3 切换时立即在LCD上显示对应槽位的Logo
- 2.4 界面保持纯净，只显示Logo图片，无槽位指示器或文字

### US-3: 确认选择开机Logo
作为用户，我希望按下旋钮按钮来确认当前显示的Logo作为开机Logo。

**验收标准:**
- 3.1 在Logo界面按下按钮确认当前槽位为开机Logo
- 3.2 选择结果保存到Flash，重启后保持
- 3.3 静默保存，不显示任何反馈文字（保持界面纯净）
- 3.4 开机时自动显示用户选择的Logo槽位

### US-5: 长按删除Logo
作为用户，我希望在Logo界面长按按钮来删除当前显示的Logo。

**验收标准:**
- 5.1 在Logo界面长按按钮（≥2秒）删除当前显示的Logo槽位
- 5.2 删除后清除该槽位的Flash数据（标记为无效）
- 5.3 删除后自动切换显示下一个有效Logo
- 5.4 如果删除后无有效Logo，显示默认开机画面
- 5.5 静默删除，不显示任何反馈文字

### US-6: 槽位满时自动覆盖
作为用户，当3个Logo槽位都已满时，我希望上传新Logo能自动覆盖最旧的Logo。

**验收标准:**
- 6.1 上传时检测所有槽位是否已满
- 6.2 如果槽位已满，自动删除Slot 0的Logo
- 6.3 新Logo写入Slot 0位置
- 6.4 原Slot 1和Slot 2的Logo保持不变
- 6.5 覆盖操作对用户透明，无需额外确认

### US-4: APP端槽位选择（可选）
作为用户，我希望在APP上传Logo时可以选择目标槽位。

**验收标准:**
- 4.1 APP上传界面显示3个槽位选项
- 4.2 可以选择上传到指定槽位
- 4.3 显示每个槽位的状态（空/已占用）

## 技术约束

### 存储约束
- Flash芯片：W25Q128（16MB）
- 每个Logo数据大小：115200字节（240x240x2 RGB565）
- 每个槽位预留空间：128KB（含头部信息）
- 总占用空间：384KB（3个槽位）

### 兼容性约束
- 必须保持现有上传协议的向后兼容性
- 不影响其他模块的Flash存储区域
- 不影响现有的开机动画逻辑

### 性能约束
- Logo切换响应时间 < 500ms
- 槽位有效性检查时间 < 50ms
