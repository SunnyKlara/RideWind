# 蓝牙传输稳定性优化 - 任务列表

> **功能名称**: JDY-08蓝牙模块Logo传输协议优化  
> **版本**: v1.0  
> **创建日期**: 2026-01-17

---

## 任务概览

本任务列表分为3个阶段,共计约7天工作量:
- **阶段1**: 核心优化 (3天) - 滑动窗口 + 批量ACK
- **阶段2**: 增强功能 (2天) - 自适应速率 + 断点续传
- **阶段3**: 测试优化 (2天) - 多场景测试 + 性能调优

---

## 阶段1: 核心优化 (3天)

### 1. APP端核心数据结构实现

- [ ] 1.1 创建滑动窗口类 `SlidingWindow`
  - 实现窗口参数管理 (windowSize, sendBase, nextSeqNum)
  - 实现在途包管理 (inFlightPackets Map)
  - 实现已确认包集合 (ackedPackets Set)
  - 实现丢失包集合 (lostPackets Set)
  - 实现窗口状态查询 (isFull, isEmpty, inFlightCount)

- [ ] 1.2 创建包信息类 `PacketInfo`
  - 包含序号、数据、发送时间、重传次数、确认状态
  - 实现age计算 (当前时间 - 发送时间)
  - 实现超时判断 (isTimeout方法)

- [ ] 1.3 创建ACK信息类 `AckInfo`
  - 支持累积ACK、选择性ACK、NAK三种类型
  - 实现SACK bitmap解析 (getLostPackets方法)
  - 记录接收时间用于RTT计算

- [ ] 1.4 创建传输统计类 `TransmissionStats`
  - 统计总包数、发送包数、重传包数、丢失包数
  - 计算传输时间、平均RTT、丢包率、吞吐量
  - 实现toString方法输出详细报告

### 2. APP端滑动窗口算法实现

- [ ] 2.1 实现滑动窗口发送逻辑
  - 实现窗口内批量发送 (sendBatch方法)
  - 实现窗口滑动逻辑 (slideWindow方法)
  - 实现窗口满判断和等待机制
  - 实现发送进度跟踪

- [ ] 2.2 实现累积ACK处理
  - 解析LOGO_ACK:seq响应
  - 确认所有 <= seq 的包
  - 滑动窗口到seq+1
  - 更新RTT估算
  - 重置重复ACK计数

- [ ] 2.3 实现选择性ACK (SACK) 处理
  - 解析LOGO_SACK:seq:bitmap响应
  - 根据bitmap标记已收到和丢失的包
  - 更新sendBase到最小未确认包
  - 将丢失的包加入重传队列

- [ ] 2.4 实现重复ACK检测 (快速重传)
  - 跟踪最后ACK序号和重复次数
  - 收到3次重复ACK时触发快速重传
  - 重传nextSeq包而不等待超时

### 3. APP端超时重传机制

- [ ] 3.1 实现RTT估算器 `RTTEstimator`
  - 维护estimatedRTT和devRTT
  - 实现updateRTT方法 (平滑算法)
  - 实现getTimeout方法 (EstimatedRTT + 4*DevRTT)
  - 限制超时范围 (300-3000ms)

- [ ] 3.2 实现超时检测和重传
  - 定期检查在途包是否超时
  - 超时包增加重传次数
  - 超过最大重传次数抛出异常
  - 实现指数退避 (增加超时时间)

- [ ] 3.3 实现重传队列管理
  - 维护需要重传的包列表
  - 按优先级排序 (最早的包优先)
  - 实现retransmitPacket方法
  - 实现retransmitOldest方法

### 4. 硬件端接收窗口实现

- [ ] 4.1 修改logo.c添加接收窗口结构
  - 定义ReceiveWindow_t结构体
  - 添加expectedSeq, maxReceivedSeq字段
  - 添加receiveBitmap数组 (256 bits)
  - 添加缓冲区管理字段

- [ ] 4.2 实现位图操作函数
  - 实现Bitmap_Set (设置某位为1)
  - 实现Bitmap_IsSet (检查某位是否为1)
  - 实现Bitmap_Clear (清空位图)
  - 实现Bitmap_GenerateSACK (生成16位bitmap字符串)

- [ ] 4.3 修改LOGO_DATA处理逻辑
  - 检查包是否已接收 (幂等处理)
  - 使用位图记录接收状态
  - 更新最大连续序号
  - 实现累积ACK发送 (每10包)
  - 检测丢包并发送SACK

- [ ] 4.4 添加LOGO_QUERY_PROGRESS命令支持
  - 响应进度查询请求
  - 返回lastAckedSeq和totalPackets
  - 用于断点续传

### 5. 集成测试和调试

- [ ] 5.1 单元测试
  - 测试滑动窗口基本功能
  - 测试ACK处理逻辑
  - 测试RTT估算准确性
  - 测试位图操作正确性

- [ ] 5.2 集成测试 - 正常环境
  - 测试完整传输流程
  - 验证传输时间 < 50秒
  - 验证成功率 > 95%
  - 验证丢包率 < 5%

- [ ] 5.3 调试和优化
  - 添加详细日志输出
  - 分析传输瓶颈
  - 优化窗口大小参数
  - 优化ACK频率

## 阶段2: 增强功能 (2天)

### 6. 自适应速率控制

- [ ] 6.1 实现丢包率监控 `PacketLossMonitor`
  - 统计发送包数、丢失包数、重传包数
  - 计算实时丢包率
  - 实现每100包重置统计

- [ ] 6.2 实现自适应速率控制器 `AdaptiveRateController`
  - 维护当前发送间隔 (初始20ms)
  - 根据丢包率动态调整间隔
  - 丢包率<5%: 加速到10ms
  - 丢包率>15%: 减速到80ms
  - 实现waitBeforeSend方法

- [ ] 6.3 实现窗口大小动态调整 `WindowSizeController`
  - 维护当前窗口大小 (初始10)
  - 跟踪连续成功/失败次数
  - 连续10次成功: 增大窗口
  - 连续3次失败: 减小窗口
  - 限制窗口范围 (5-20)

- [ ] 6.4 集成速率控制到传输流程
  - 在发送循环中调用waitBeforeSend
  - 定期更新丢包率统计
  - 定期调整速率和窗口大小
  - 记录调整日志

### 7. 断点续传功能

- [ ] 7.1 实现传输进度类 `TransmissionProgress`
  - 记录totalPackets, lastAckedSeq
  - 记录receivedPackets集合
  - 记录lastUpdateTime
  - 实现save方法 (保存到SharedPreferences)
  - 实现load方法 (从本地加载)
  - 实现clear方法 (清除进度)

- [ ] 7.2 实现断点恢复流程
  - 查询硬件端进度 (LOGO_QUERY_PROGRESS)
  - 加载本地进度
  - 取两者最小值作为断点
  - 从断点继续传输
  - 处理硬件端不支持的情况

- [ ] 7.3 修改传输流程支持断点续传
  - 在开始传输前检查是否有断点
  - 询问用户是否从断点继续
  - 调整窗口起始位置
  - 定期保存进度

- [ ] 7.4 测试断点续传功能
  - 测试传输中断后恢复
  - 测试进度保存和加载
  - 测试超时进度自动清除
  - 测试多设备独立进度

### 8. 性能优化

- [ ] 8.1 实现批量发送优化
  - 实现sendBatch方法
  - 批量构建命令字符串
  - 减少异步调用开销

- [ ] 8.2 实现ACK合并处理 `AckBatcher`
  - 延迟50ms批量处理ACK
  - 合并多个累积ACK
  - 批量处理SACK
  - 减少UI更新频率

- [ ] 8.3 实现内存优化
  - 创建PacketPool对象池
  - 复用Uint8List对象
  - 减少GC压力

- [ ] 8.4 实现日志优化 `TransmissionLogger`
  - 每50包输出一次日志
  - 重要事件立即输出
  - 减少串口输出影响

## 阶段3: 测试优化 (2天)

### 9. 错误处理和恢复

- [ ] 9.1 定义错误类型枚举 `TransmissionError`
  - 连接错误 (bluetoothDisconnected, connectionTimeout)
  - 协议错误 (invalidResponse, protocolMismatch)
  - 传输错误 (packetLost, checksumMismatch, timeoutExceeded)
  - 硬件错误 (flashEraseFailed, flashWriteFailed, bufferOverflow)
  - 其他错误 (userCancelled, unknownError)

- [ ] 9.2 实现传输异常类 `TransmissionException`
  - 包含错误类型、消息、详情、是否可恢复
  - 实现toString方法

- [ ] 9.3 实现错误处理函数 `handleError`
  - 针对每种错误类型实现恢复策略
  - 蓝牙断开: 等待重连 + 断点续传
  - 丢包: 重传
  - 超时: 降速重试
  - 校验失败: 完全重传
  - 用户取消: 清理资源

- [ ] 9.4 实现重试策略 `RetryPolicy`
  - 实现指数退避算法
  - 限制最大重试次数
  - 实现retryWithPolicy包装函数
