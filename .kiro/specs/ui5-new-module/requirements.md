# UI5 新模块控制界面 - 需求规格

📅 创建日期: 2026-01-14
🎯 目标: 在 STM32 LCD 屏幕上新增 UI=5 界面，用于控制新的外部模块

---

## 1. 背景与目标

### 1.1 项目背景

RideWind 硬件系统当前支持 5 个 LCD 界面 (UI 0-4)：
- UI=0: 开机动画
- UI=1: 风扇调速界面 (Running Mode)
- UI=2: 配色预设界面 (Colorize Mode)
- UI=3: RGB 调色界面
- UI=4: 亮度调节界面

现有 LCD_ui5 是蓝牙调试界面（未使用），可以被替换为新的功能界面。

### 1.2 开发目标

创建一个新的 UI=5 界面，用于控制新的外部模块。该界面应：
1. **视觉设计**: 完全复制 UI=1 (风扇调速界面) 的视觉样式
2. **控制逻辑**: 独立的控制变量和 PWM 输出通道
3. **蓝牙协议**: 支持 APP 远程控制和状态同步

## 2. 用户故事

### US-001: 界面切换
**作为** 用户

- [ ] 硬件状态变化时主动上报给 APP
- [ ] APP 与硬件状态保持同步

### US-004: 单位切换 (可选)
**作为** 用户

## 3. 功能需求 (EARS 格式)

### FR-001: 界面显示
**When** 用户切换到 UI=5 界面
**the system shall** 显示与 UI=1 相同布局的控制界面，包括：
- 背景图片 (gImage_beijing_240_240)
- 数值显示区域 (大字体数字)
- 单位/状态指示区域

### FR-002: 编码器控制
**While** 在 UI=5 界面
**when** 用户旋转编码器
**the system shall** 调整新模块的控制变量 (Module5_Num)，范围 0-100

### FR-003: PWM 输出
**While** 在 UI=5 界面
**when** 控制变量 (Module5_Num) 变化
**the system shall** 更新对应 PWM 通道的占空比，映射关系: 0-100 → 0-1000

### FR-004: 蓝牙命令接收
**When** 收到 `MOD5:xx` 命令 (xx = 0-100)
**the system shall** 设置 Module5_Num 为指定值，并更新 PWM 输出

### FR-005: 状态查询
**When** 收到 `GET:MOD5` 命令
**the system shall** 响应 `MOD5:xx\r\n`，返回当前 Module5_Num 值

### FR-006: 状态上报
**When** Module5_Num 值因本地操作变化
**the system shall** 主动发送 `MOD5_REPORT:xx\n` 到 APP

### FR-007: UI 命令支持
**When** 收到 `UI:5` 命令
**the system shall** 切换到 UI=5 界面并初始化显示

## 4. 硬件接口需求

### 4.1 可用引脚 (待确认)

根据电路原理图，以下引脚可用于新模块：

| 引脚 | 功能 | 说明 |
|------|------|------|
| PC8 | TIM_CH1 | 可用 PWM 输出 |
| PC12 | DC2 | 可用 GPIO/PWM |
| WAFER 连接器 | 扩展接口 | 外部模块连接 |

### 4.2 PWM 配置 (待确认)

**推荐配置**:
- 定时器: TIM8 或 TIM3 (需确认可用通道)
- PWM 频率: 1kHz (与风扇一致)
- 分辨率: 0-999 (Period=1000-1)
