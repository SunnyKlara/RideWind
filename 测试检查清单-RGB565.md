## 预期LCD显示内容

### 阶段1: 初始化 (0-5秒)
```
LOGO DEBUG
====================
START CMD RCV
Size:115200 CRC:0
ERASING...
ERASE DONE
READY RCV 7200 PKT
```

**检查点**:
- ✅ 显示"Size:115200"（不是1024）
- ✅ 显示"READY RCV 7200 PKT"（不是64）
- ✅ 没有"SIZE MISMATCH!"错误

### 阶段2: 传输中 (5秒-6分钟)
```
RCV:100/7200
RCV:200/7200
RCV:300/7200
...
RCV:7100/7200
RCV:7200/7200
```

**检查点**:
- ✅ 进度每100包更新一次
- ✅ 总包数是7200（不是64）
- ✅ 没有"LOGO_BUSY"或"LOGO_NAK"错误

### 阶段3: 验证 (6分钟后)
```
END CMD RCV
VERIFYING...
SIZE CHECK OK        ← 关键！
CALC CRC32...
CRC32 OK/FAIL       ← 可能失败
WRITE HEADER...
UPLOAD OK!          ← 如果CRC32通过
```

**检查点**:
- ✅ 显示"SIZE CHECK OK"（不是"SIZE MISMATCH!"）
- ⚠️ CRC32可能显示"FAIL"（当前硬编码为0）
- ✅ 如果CRC32通过，显示"UPLOAD OK!"

## 故障排查

### 问题: LCD没有显示调试信息
**原因**: 可能没有自动切换到Logo界面  
**解决**: 手动切换到Logo界面（ui=6）

### 问题: 显示"SIZE MISMATCH!"
**原因**: 修复未生效  
**检查**:
1. 确认E2E测试文件已更新
2. 重新编译APP: `flutter clean && flutter run`
3. 检查日志中的数据大小

### 问题: 显示"CRC32 FAIL!"
**原因**: CRC32当前硬编码为0  
**说明**: 这是预期的，下一步修复  
**重要**: 只要"SIZE CHECK OK"就说明本次修复成功

### 问题: 传输很慢
**原因**: 7200包需要约6分钟  
**说明**: 这是正常的（之前64包只需3秒）  
**优化**: 后续可以增加ACK频率或包大小

## 技术参数

### RGB565格式
- 每像素: 2字节
- 红色: 5位 (0-31)
- 绿色: 6位 (0-63)
- 蓝色: 5位 (0-31)
- 字节序: 大端序

### 传输参数
- 每包大小: 16字节
- 总包数: 7,200包
- ACK频率: 每100包
- 总ACK数: 72次
- 预计时间: 约6分钟

### Flash存储
- 起始地址: 0x100000 (1MB位置)
- 头部大小: 16字节
- 数据大小: 115,200字节
- 总大小: 115,216字节
- 扇区数: 28个 (每个4KB)
