## 🐛 故障排查

### 问题1: LCD没有切换到Logo界面
**可能原因**:
- 固件没有烧录最新版本
- LOGO命令没有被接收

**检查方法**:
- 查看串口日志，看是否有 `[LOGO] ParseCommand: LOGO_START:...`
- 确认APP显示"已连接"

### 问题2: 切换到Logo界面但没有调试信息
**可能原因**:
- 协议格式还是不对
- 命令没有被正确解析

**检查方法**:
- 查看串口日志
- 确认APP发送的命令格式是 `LOGO_START:1024:0`

### 问题3: 显示"大小不匹配"
**可能原因**:
- 硬件期望的大小不是1024

**解决方法**:
- 检查 `LOGO_DATA_SIZE` 的定义
- 确认是128x64单色位图（1024字节）

### 问题4: CRC32校验失败
**这是预期的！** 因为START命令中CRC设置为0：
```dart
final command = 'LOGO_START:$dataSize:0\n';  // CRC=0
```

实际数据的CRC32不是0，所以会失败。但这不影响测试目的。
